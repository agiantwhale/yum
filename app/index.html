1. Wtf is this?

Some girl posted on the University of Michigan Class of 2019 Facebook page that she intended to visit every restaurant in Ann Arbor. Since I have way too much free time on my hands I decided to write a software that can calculate the shortest route one would need to take to visit every restaurant in Ann Arbor.

2. What's with all the lines? Why is it so slow?

Good question! Finding the SHORTEST route given a set of N points is a very complicated math problem - nicknamed "Travelling Salesman" problem with NP-Complete complexity. It's a topic that spans the fields of mathematics, computer science and game theory. So instead, the approach that I took over here is simulating an evolutionary process. Start with a terrible (but correct) parent route, mutate that route, out of offspring routes select 4 best routes, repeat this iteration hundreds of times until we get to a acceptable solution.
